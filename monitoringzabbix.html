Monitoring avec Zabbix

Solution de supervision d'infrastructure open-source

Hugo Lemarchand · Ensitech · Session 2026

Présentation

Zabbix est une solution de monitoring open-source permettant de surveiller en temps réel des serveurs, équipements réseau, applications, bases de données et environnements cloud.

Ce projet correspond à la mise en place d'une supervision complète en entreprise, incluant l'installation du serveur, le déploiement des agents, la configuration des alertes et la création de dashboards.

Architecture Zabbix

ComposantRôle

Zabbix FrontendInterface web de gestion
Zabbix AgentCollecteur installé sur les hôtes surveillés
Zabbix ProxyCollecteur intermédiaire (optionnel)
DatabaseStockage des données (MySQL / PostgreSQL)

Installation du serveur Zabbix

1

Ajouter le dépôt Zabbix

# Pour Debian 12
wget https://repo.zabbix.com/zabbix/7.0/debian/pool/main/z/zabbix-release/zabbix-release_7.0-1+debian12_all.deb
sudo dpkg -i zabbix-release_7.0-1+debian12_all.deb
sudo apt update
2

Installer les composants

sudo apt install -y zabbix-server-mysql zabbix-frontend-php \
  zabbix-apache-conf zabbix-sql-scripts zabbix-agent
3

Créer la base de données

sudo mysql -u root -p
CREATE DATABASE zabbix CHARACTER SET utf8mb4 COLLATE utf8mb4_bin;
CREATE USER 'zabbix'@'localhost' IDENTIFIED BY 'MotDePasseZabbix123!';
GRANT ALL PRIVILEGES ON zabbix.* TO 'zabbix'@'localhost';
SET GLOBAL log_bin_trust_function_creators = 1;
FLUSH PRIVILEGES;
EXIT;
4

Importer le schéma de base

zcat /usr/share/zabbix-sql-scripts/mysql/server.sql.gz | \
  mysql --default-character-set=utf8mb4 -uzabbix -p zabbix

# Désactiver log_bin_trust_function_creators
sudo mysql -u root -p -e "SET GLOBAL log_bin_trust_function_creators = 0;"
5

Configurer Zabbix Server

sudo nano /etc/zabbix/zabbix_server.conf

# Modifier :
DBPassword=MotDePasseZabbix123!
6

Configurer PHP et démarrer les services

sudo nano /etc/zabbix/apache.conf
# Décommenter et définir :
php_value date.timezone Europe/Paris
sudo systemctl restart zabbix-server zabbix-agent apache2
sudo systemctl enable zabbix-server zabbix-agent apache2

Configuration initiale (interface web)

7

Accéder à l'interface

http://IP_DU_SERVEUR/zabbix
8

Assistant d'installation

  1. Vérifier les prérequis (tout en vert)
  2. Configurer la connexion DB : type MySQL, host localhost, base zabbix, user zabbix
  3. Configurer le nom du serveur Zabbix
  4. Finaliser l'installation

Identifiants par défaut : login Admin / mot de passe zabbix

⚠️ Important Changer immédiatement le mot de passe Admin après la première connexion !

Installation des agents

Agent Linux (Debian/Ubuntu)

wget https://repo.zabbix.com/zabbix/7.0/debian/pool/main/z/zabbix-release/zabbix-release_7.0-1+debian12_all.deb
sudo dpkg -i zabbix-release_7.0-1+debian12_all.deb
sudo apt update && sudo apt install zabbix-agent2 -y

sudo nano /etc/zabbix/zabbix_agent2.conf
# Modifier :
Server=IP_SERVEUR_ZABBIX
ServerActive=IP_SERVEUR_ZABBIX
Hostname=nom-du-serveur

sudo systemctl restart zabbix-agent2 && sudo systemctl enable zabbix-agent2

Agent Windows

msiexec /i zabbix_agent2-7.0.0-windows-amd64.msi /qn SERVER=IP_SERVEUR HOSTNAME=PC-WIN01

Ou configurer manuellement : C:\Program Files\Zabbix Agent 2\zabbix_agent2.conf

Pare-feu

sudo ufw allow 10051/tcp   # Serveur Zabbix (trapper)
sudo ufw allow 10050/tcp   # Agent Zabbix

Ajout d'hôtes à superviser

9

Via l'interface web

  1. Configuration → Hosts → Create host
  2. Renseigner le Host name, le groupe et l'interface (IP + port 10050)
  3. Onglet Templates : associer un template de monitoring

Configuration des alertes

10

Configurer les médias de notification

# Administration → Media types → Email
SMTP server      : smtp.gmail.com
SMTP server port : 587
SMTP email       : votreemail@gmail.com
Authentication   : Username and password

Pour Discord ou Slack : utiliser les templates de webhook intégrés ou un script personnalisé.

11

Créer des actions

  1. Configuration → Actions → Trigger actions → Create action
  2. Conditions : définir quand déclencher (ex: severity ≥ Warning)
  3. Operations : envoyer notification aux utilisateurs ou groupes concernés
💡 Astuce Configurer des escalades pour les problèmes non résolus après un certain délai.

Éléments à surveiller en entreprise

Checklist de supervision
  • CPU, RAM, disque des serveurs
  • Services critiques (AD, DNS, DHCP)
  • Disponibilité des sites web
  • État des sauvegardes
  • Trafic réseau des switches
  • Expiration des certificats SSL
  • État des VMs (VMware, Proxmox)
  • Performances des bases de données
