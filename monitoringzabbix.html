📊 Monitoring avec Zabbix

Solution de supervision d'infrastructure complète

📋 Présentation

Zabbix est une solution de monitoring open-source permettant de surveiller :

  • Serveurs (Linux, Windows)
  • Équipements réseau (switches, routeurs)
  • Applications et services
  • Bases de données
  • Cloud et virtualisation

Architecture Zabbix

Composant Rôle
Zabbix Server Collecte et traitement des données
Zabbix Frontend Interface web de gestion
Zabbix Agent Collecteur sur les hôtes surveillés
Zabbix Proxy Collecteur intermédiaire (optionnel)
Database Stockage des données (MySQL/PostgreSQL)

🔧 Installation du serveur Zabbix

1Ajouter le dépôt Zabbix

# Pour Debian 12
wget [repo.zabbix.com](https://repo.zabbix.com/zabbix/7.0/debian/pool/main/z/zabbix-release/zabbix-release_7.0-1+debian12_all.deb)
sudo dpkg -i zabbix-release_7.0-1+debian12_all.deb
sudo apt update

2Installer les composants

sudo apt install -y zabbix-server-mysql zabbix-frontend-php zabbix-apache-conf zabbix-sql-scripts zabbix-agent

3Créer la base de données

sudo mysql -u root -p
CREATE DATABASE zabbix CHARACTER SET utf8mb4 COLLATE utf8mb4_bin;
CREATE USER 'zabbix'@'localhost' IDENTIFIED BY 'MotDePasseZabbix123!';
GRANT ALL PRIVILEGES ON zabbix.* TO 'zabbix'@'localhost';
SET GLOBAL log_bin_trust_function_creators = 1;
FLUSH PRIVILEGES;
EXIT;

4Importer le schéma de base

zcat /usr/share/zabbix-sql-scripts/mysql/server.sql.gz | mysql --default-character-set=utf8mb4 -uzabbix -p zabbix
# Désactiver log_bin_trust_function_creators
sudo mysql -u root -p -e "SET GLOBAL log_bin_trust_function_creators = 0;"

5Configurer Zabbix Server

sudo nano /etc/zabbix/zabbix_server.conf
DBPassword=MotDePasseZabbix123!

6Configurer PHP

sudo nano /etc/zabbix/apache.conf

# Décommenter et définir :
php_value date.timezone Europe/Paris

7Démarrer les services

sudo systemctl restart zabbix-server zabbix-agent apache2
sudo systemctl enable zabbix-server zabbix-agent apache2

🖥️ Configuration initiale (interface web)

8Accéder à l'interface

[ip_du_serveur](http://IP_DU_SERVEUR/zabbix)

9Assistant d'installation

  1. Vérifier les prérequis (tout en vert)
  2. Configurer la connexion base de données :
    • Database type : MySQL
    • Database host : localhost
    • Database name : zabbix
    • User : zabbix
    • Password : MotDePasseZabbix123!
  3. Configurer le nom du serveur Zabbix
  4. Finaliser l'installation

Identifiants par défaut

  • Login : Admin
  • Mot de passe : zabbix
⚠️ Important : Changer immédiatement le mot de passe Admin !

🔌 Installation des agents

Agent Linux (Debian/Ubuntu)

# Ajouter le dépôt
wget [repo.zabbix.com](https://repo.zabbix.com/zabbix/7.0/debian/pool/main/z/zabbix-release/zabbix-release_7.0-1+debian12_all.deb)
sudo dpkg -i zabbix-release_7.0-1+debian12_all.deb
sudo apt update

# Installer l'agent
sudo apt install zabbix-agent2 -y

# Configurer
sudo nano /etc/zabbix/zabbix_agent2.conf
Server=IP_SERVEUR_ZABBIX
ServerActive=IP_SERVEUR_ZABBIX
Hostname=nom-du-serveur
# Démarrer et activer
sudo systemctl restart zabbix-agent2
sudo systemctl enable zabbix-agent2

Agent Windows

  1. Télécharger l'agent depuis zabbix.com
  2. Installation via MSI : 
    msiexec /i zabbix_agent2-7.0.0-windows-amd64.msi /qn SERVER=IP_SERVEUR HOSTNAME=PC-WIN01
  3. Ou installer manuellement et configurer C:\Program Files\Zabbix Agent 2\zabbix_agent2.conf

Configuration du pare-feu

# Sur le serveur Zabbix
sudo ufw allow 10051/tcp  # Zabbix trapper

# Sur les agents
sudo ufw allow 10050/tcp  # Zabbix agent

➕ Ajout d'hôtes à superviser

10Via l'interface web

  1. Configuration → Hosts → Create host
  2. Configurer :
    • Host name : nom de l'hôte
    • Groups : sélectionner ou créer un groupe
    • Interfaces : ajouter l'IP et le port (10050)
  3. Onglet Templates : associer des templates de monitoring

Templates recommandés

Type d'hôte Template
Linux Linux by Zabbix agent
Windows Windows by Zabbix agent
Switch/Routeur Template Network Generic Device SNMPv2
VMware VMware by HTTP
MySQL MySQL by Zabbix agent
Apache Apache by Zabbix agent

Webhook Discord/Slack

Utiliser les templates de webhook intégrés ou créer un script personnalisé.

12Créer des actions

  1. Configuration → Actions → Trigger actions → Create action
  2. Conditions : définir quand déclencher (ex: severity >= Warning)
  3. Operations : envoyer notification aux utilisateurs/groupes
💡 Astuce : Configurer des escalades pour les problèmes non résolus après un certain temps.

📈 Création de dashboards

13Dashboard personnalisé

  1. Monitoring → Dashboards → Create dashboard
  2. Ajouter des widgets :
    • Problems : liste des problèmes actifs
    • Graph : graphiques de métriques
    • Host availability : état des hôtes
    • System information : infos serveur Zabbix
    • Map : carte du réseau

Créer une carte réseau

  1. Monitoring → Maps → Create map
  2. Ajouter des éléments (hôtes, groupes)
  3. Créer des liens entre les éléments
  4. Les liens changent de couleur selon l'état
