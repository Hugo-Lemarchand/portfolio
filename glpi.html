Installation GLPI

Gestion Libre de Parc Informatique – Helpdesk & Inventaire

Hugo Lemarchand · Ensitech · Session 2026

Présentation

GLPI est une solution open-source de gestion de parc informatique et de helpdesk. Elle permet la gestion des tickets d'incident, l'inventaire du matériel et des logiciels, ainsi que le suivi des contrats et des budgets IT.

Ce déploiement s'inscrit dans mon stage SISR, où GLPI a été utilisé pour la gestion des demandes utilisateurs et l'inventaire du parc, en interface avec OCS Inventory.

Prérequis

  • Serveur Debian/Ubuntu avec Apache2 installé
  • PHP 8.1 ou supérieur
  • MariaDB / MySQL
  • Minimum 2 Go de RAM, 2 Go d'espace disque

Installation des dépendances

1

Installer PHP et les extensions requises

sudo apt update
sudo apt install -y php php-{curl,gd,imagick,intl,apcu,memcache,imap,mysql,ldap,tidy,xmlrpc,mbstring,json,iconv,xml,xsl,zip,bz2}
2

Installer MariaDB

sudo apt install mariadb-server -y
sudo mysql_secure_installation
3

Créer la base de données

sudo mysql -u root -p
CREATE DATABASE glpi CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'glpi_user'@'localhost' IDENTIFIED BY 'MotDePasseFort123!';
GRANT ALL PRIVILEGES ON glpi.* TO 'glpi_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;

Téléchargement et installation

4

Télécharger GLPI

cd /tmp
wget https://github.com/glpi-project/glpi/releases/download/10.0.16/glpi-10.0.16.tgz
tar -xvzf glpi-10.0.16.tgz
sudo mv glpi /var/www/html/
5

Configurer les permissions

sudo chown -R www-data:www-data /var/www/html/glpi
sudo chmod -R 755 /var/www/html/glpi
6

Sécuriser les dossiers sensibles

sudo mkdir -p /etc/glpi /var/lib/glpi /var/log/glpi
sudo mv /var/www/html/glpi/config /etc/glpi/
sudo mv /var/www/html/glpi/files /var/lib/glpi/
sudo chown -R www-data:www-data /etc/glpi /var/lib/glpi /var/log/glpi
7

Fichiers de configuration

sudo nano /var/www/html/glpi/inc/downstream.php
<?php
define('GLPI_CONFIG_DIR', '/etc/glpi/');
if (file_exists(GLPI_CONFIG_DIR . '/local_define.php')) {
    require_once GLPI_CONFIG_DIR . '/local_define.php';
}
sudo nano /etc/glpi/local_define.php
<?php
define('GLPI_VAR_DIR', '/var/lib/glpi');
define('GLPI_LOG_DIR', '/var/log/glpi');

Configuration Apache

sudo nano /etc/apache2/sites-available/glpi.conf
<VirtualHost *:80>
    ServerName glpi.mondomaine.fr
    DocumentRoot /var/www/html/glpi/public

    <Directory /var/www/html/glpi/public>
        Require all granted
        RewriteEngine On
        RewriteCond %{REQUEST_FILENAME} !-f
        RewriteRule ^(.*)$ index.php [QSA,L]
    </Directory>

    ErrorLog ${APACHE_LOG_DIR}/glpi_error.log
    CustomLog ${APACHE_LOG_DIR}/glpi_access.log combined
</VirtualHost>
sudo a2ensite glpi.conf
sudo a2enmod rewrite
sudo systemctl restart apache2

Installation via l'interface web

8

Accéder à l'assistant

Ouvrir un navigateur et accéder à :

http://IP_DU_SERVEUR/glpi
9

Étapes de l'assistant

  1. Sélectionner la langue (Français)
  2. Accepter la licence GPL
  3. Choisir "Installer"
  4. Vérifier les prérequis (tout doit être en vert)
  5. Configurer la connexion à la base de données (host : localhost, user : glpi_user)
  6. Sélectionner la base glpi
  7. Finaliser l'installation

Sécurisation post-installation

# Supprimer le fichier d'installation
sudo rm /var/www/html/glpi/install/install.php

# Changer les mots de passe par défaut
# Administration → Utilisateurs → Modifier chaque utilisateur

# Configurer les tâches automatiques (cron)
sudo crontab -u www-data -e
# Ajouter :
*/1 * * * * php /var/www/html/glpi/front/cron.php

Intégration avec OCS Inventory

Interface GLPI ↔ OCS
  1. Configuration → Plugins → Installer le plugin OCS
  2. Configurer la connexion au serveur OCS
  3. Planifier les imports automatiques
