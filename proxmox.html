Installation Proxmox VE

Hyperviseur de virtualisation open-source – Guide d'installation et de configuration

Hugo Lemarchand · Ensitech · Session 2026

Présentation

Ce tutoriel présente l'installation et la configuration de Proxmox Virtual Environment (VE), un hyperviseur open-source basé sur Debian permettant de gérer des machines virtuelles (KVM) et des conteneurs (LXC) depuis une interface web centralisée.

Ce projet s'inscrit dans mon parcours SISR, en lien direct avec la virtualisation d'infrastructure et l'administration de serveurs. Proxmox a été utilisé dans le cadre de mes environnements de test et de déploiement en laboratoire.

Prérequis matériels et logiciels

🖥️ Processeur Compatible virtualisation VT-x / AMD-V requis
🧠 RAM 4 Go minimum 8 Go+ recommandé
💾 Stockage 32 Go minimum SSD recommandé
🔌 Clé USB 8 Go minimum Pour l'ISO bootable
📀 ISO Proxmox VE Depuis proxmox.com Version stable

Étapes d'installation

1

Création de la clé USB bootable

Télécharger l'ISO depuis le site officiel, puis flasher la clé avec Rufus, Balena Etcher, ou dd sous Linux :

sudo dd if=proxmox-ve_*.iso of=/dev/sdX bs=4M status=progress
2

Démarrage sur la clé USB

  • Accéder au BIOS/UEFI et définir la clé USB comme périphérique de démarrage prioritaire
  • Activer la virtualisation matérielle (VT-x ou AMD-V)
  • Sélectionner Install Proxmox VE dans le menu de démarrage
3

Configuration de l'installation

  • Accepter la licence EULA
  • Sélectionner le disque cible pour l'installation
  • Configurer le pays, le fuseau horaire et la disposition du clavier
  • Définir un mot de passe root sécurisé et une adresse e-mail administrateur
4

Configuration réseau

Renseigner les paramètres réseau du serveur :

Hostname:   proxmox.local
IP Address: 192.168.1.100
Netmask:    255.255.255.0
Gateway:    192.168.1.1
DNS Server: 192.168.1.1
5

Finalisation et premier accès

Une fois l'installation terminée et le serveur redémarré, accéder à l'interface web via un navigateur :

https://192.168.1.100:8006
💡 Note Le navigateur affichera un avertissement de certificat auto-signé. Cliquer sur "Avancé" puis "Continuer" pour accéder à l'interface.

Configuration post-installation

Désactiver le dépôt Enterprise (optionnel)

Si vous n'avez pas de licence commerciale, désactivez le dépôt Enterprise pour éviter les erreurs lors des mises à jour :

# Commenter la ligne dans le fichier sources
nano /etc/apt/sources.list.d/pve-enterprise.list

# Ajouter le dépôt no-subscription
echo "deb http://download.proxmox.com/debian/pve bookworm pve-no-subscription" \
  > /etc/apt/sources.list.d/pve-no-subscription.list

# Mettre à jour le système
apt update && apt full-upgrade -y

Configurer un stockage pour les VMs

  • Datacenter → Storage → Add
  • Choisir le type adapté : Directory, LVM, ZFS…
  • Définir le chemin de stockage et les types de contenus autorisés
💡 Conseil ZFS est recommandé pour les environnements critiques : il offre snapshots, compression et réplication native.

Création de votre première VM

Procédure via l'interface web
  1. Cliquer sur Create VM en haut à droite de l'interface
  2. Définir l'ID et le nom de la VM
  3. Sélectionner l'image ISO à utiliser (depuis le stockage)
  4. Configurer le disque virtuel : taille et format (qcow2 ou raw)
  5. Allouer les ressources CPU et RAM
  6. Configurer le réseau sur le bridge vmbr0
  7. Valider, puis démarrer la VM depuis le panneau de gauche

Commandes utiles

# Lister les VMs
qm list

# Démarrer une VM
qm start <vmid>

# Arrêter une VM
qm stop <vmid>

# Créer un snapshot
qm snapshot <vmid> <snapname>

# Consulter les logs du service Proxmox
journalctl -u pvedaemon
