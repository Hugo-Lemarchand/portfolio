🌐 Installation Apache 2

Serveur web HTTP open-source

📋 Prérequis

  • Serveur Debian/Ubuntu à jour
  • Accès root ou sudo
  • Port 80 et 443 disponibles
  • Connexion internet pour télécharger les paquets

🔧 Installation

1Mise à jour du système

sudo apt update && sudo apt upgrade -y

2Installation d'Apache

sudo apt install apache2 -y

3Vérification du service

# Vérifier le statut
sudo systemctl status apache2

# Activer au démarrage
sudo systemctl enable apache2

# Tester dans un navigateur
[ip_du_serveur](http://IP_DU_SERVEUR)

📁 Structure des fichiers

/etc/apache2/ ├── apache2.conf # Configuration principale ├── ports.conf # Ports d'écoute ├── sites-available/ # Sites disponibles ├── sites-enabled/ # Sites activés (liens symboliques) ├── mods-available/ # Modules disponibles ├── mods-enabled/ # Modules activés └── conf-available/ # Configurations additionnelles /var/www/html/ # Racine web par défaut /var/log/apache2/ # Logs (access.log, error.log)

⚙️ Configuration d'un Virtual Host

1Créer le répertoire du site

sudo mkdir -p /var/www/monsite.fr/public_html
sudo chown -R $USER:$USER /var/www/monsite.fr
sudo chmod -R 755 /var/www/monsite.fr

2Créer une page de test

nano /var/www/monsite.fr/public_html/index.html
<!DOCTYPE html>
<html>
<head>
    <title>Bienvenue sur MonSite</title>
</head>
<body>
    <h1>Le site monsite.fr fonctionne !</h1>
</body>
</html>

3Créer le fichier de configuration

sudo nano /etc/apache2/sites-available/monsite.fr.conf
<VirtualHost *:80>
    ServerAdmin admin@monsite.fr
    ServerName monsite.fr
    ServerAlias www.monsite.fr
    DocumentRoot /var/www/monsite.fr/public_html

    <Directory /var/www/monsite.fr/public_html>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>

    ErrorLog ${APACHE_LOG_DIR}/monsite.fr_error.log
    CustomLog ${APACHE_LOG_DIR}/monsite.fr_access.log combined
</VirtualHost>

4Activer le site

# Activer le site
sudo a2ensite monsite.fr.conf

# Désactiver le site par défaut (optionnel)
sudo a2dissite 000-default.conf

# Vérifier la configuration
sudo apache2ctl configtest

# Recharger Apache
sudo systemctl reload apache2

🔒 Installation SSL avec Let's Encrypt

1Installer Certbot

sudo apt install certbot python3-certbot-apache -y

2Obtenir le certificat

sudo certbot --apache -d monsite.fr -d www.monsite.fr

3Renouvellement automatique

# Tester le renouvellement
sudo certbot renew --dry-run

# Le cron est automatiquement configuré
💡 Note : Certbot modifie automatiquement la configuration Apache pour rediriger HTTP vers HTTPS.

🔌 Modules utiles

# Activer mod_rewrite (réécriture d'URL)
sudo a2enmod rewrite

# Activer mod_ssl (HTTPS)
sudo a2enmod ssl

# Activer mod_headers (en-têtes personnalisés)
sudo a2enmod headers

# Activer mod_deflate (compression)
sudo a2enmod deflate

# Redémarrer Apache après activation
sudo systemctl restart apache2

🛡️ Sécurisation

Masquer la version d'Apache

sudo nano /etc/apache2/conf-available/security.conf

# Modifier ces lignes :
ServerTokens Prod
ServerSignature Off

Configuration des en-têtes de sécurité

# Dans le VirtualHost ou .htaccess
Header always set X-Content-Type-Options "nosniff"
Header always set X-Frame-Options "SAMEORIGIN"
Header always set X-XSS-Protection "1; mode=block"

Configurer le pare-feu

sudo ufw allow 'Apache Full'
sudo ufw status

📚 Commandes utiles

# Gestion du service
sudo systemctl start/stop/restart/reload apache2

# Activer/désactiver un site
sudo a2ensite nom_du_site.conf
sudo a2dissite nom_du_site.conf

# Activer/désactiver un module
sudo a2enmod nom_du_module
sudo a2dismod nom_du_module

# Vérifier la configuration
sudo apache2ctl configtest

# Voir les logs en temps réel
sudo tail -f /var/log/apache2/error.log
sudo tail -f /var/log/apache2/access.log
