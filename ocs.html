Installation OCS Inventory

Open Computers and Software Inventory – Inventaire automatique du parc

Hugo Lemarchand · Ensitech · Session 2026

Présentation

OCS Inventory NG est une solution open-source d'inventaire automatique du parc informatique. Elle collecte les informations matérielles et logicielles des postes via des agents déployés, et les centralise dans une interface web pour consultation et export vers GLPI.

Ce déploiement s'inscrit dans le projet de mise en place d'une solution de gestion de parc complète, couplée à GLPI pour le helpdesk.

Prérequis

  • Serveur Debian/Ubuntu
  • Apache2 avec mod_perl
  • MariaDB / MySQL
  • PHP 7.4+ avec extensions
  • Perl et modules CPAN
  • Minimum 2 Go RAM, 10 Go disque

Installation des dépendances

1

Paquets de base

sudo apt update && sudo apt upgrade -y
sudo apt install -y apache2 libapache2-mod-perl2 mariadb-server
sudo apt install -y php php-{mysql,gd,mbstring,xml,curl,zip}
sudo apt install -y perl libxml-simple-perl libdbi-perl libdbd-mysql-perl \
  libapache-dbi-perl libnet-ip-perl libsoap-lite-perl libarchive-zip-perl \
  make build-essential
2

Modules Perl additionnels

sudo cpan install XML::Entities Apache2::SOAP
sudo cpan install Mojolicious::Lite Plack::Handler Switch Archive::Zip
3

Créer la base de données

sudo mysql -u root -p
CREATE DATABASE ocsweb;
CREATE USER 'ocs_user'@'localhost' IDENTIFIED BY 'MotDePasseOCS123!';
GRANT ALL PRIVILEGES ON ocsweb.* TO 'ocs_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;

Installation du serveur OCS

4

Télécharger OCS Inventory Server

cd /tmp
wget https://github.com/OCSInventory-NG/OCSInventory-ocsreports/releases/download/2.12.1/OCSNG_UNIX_SERVER-2.12.1.tar.gz
tar -xvzf OCSNG_UNIX_SERVER-2.12.1.tar.gz
cd OCSNG_UNIX_SERVER-2.12.1
5

Lancer l'installation

sudo ./setup.sh

Répondre aux questions de l'assistant :

  • Apache config directory : /etc/apache2/conf-available
  • Apache user/group : www-data
  • Perl binary : /usr/bin/perl
  • Logfile directory : /var/log/ocsinventory-server
6

Activer la configuration Apache

sudo a2enconf ocsinventory-reports
sudo a2enconf z-ocsinventory-server
sudo a2enmod perl
sudo a2enmod rewrite
sudo systemctl restart apache2
7

Configurer les permissions

sudo chown -R www-data:www-data /var/lib/ocsinventory-reports
sudo chmod -R 755 /var/lib/ocsinventory-reports
sudo mkdir -p /var/log/ocsinventory-server
sudo chown www-data:www-data /var/log/ocsinventory-server

Configuration via l'interface web

8

Accéder à l'assistant

http://IP_DU_SERVEUR/ocsreports/install.php
9

Configurer la connexion

  • MySQL server : localhost
  • MySQL user : ocs_user
  • MySQL password : MotDePasseOCS123!
  • MySQL database : ocsweb
⚠️ Important Supprimer le fichier install.php après l'installation : 
sudo rm /usr/share/ocsinventory-reports/ocsreports/install.php

Identifiants par défaut : login admin / mot de passe admin — à changer immédiatement.

Installation de l'agent Windows

Télécharger l'agent

Depuis le GitHub officiel : OCSInventory-NG/OCSInventory-Windows-Agent

Installation silencieuse

OCS-Windows-Agent-Setup.exe /S /SERVER=http://IP_SERVEUR/ocsinventory

Fichier de configuration

Chemin : C:\ProgramData\OCS Inventory NG\Agent\ocsinventory.ini

[OCS Inventory Agent]
Server=http://IP_SERVEUR/ocsinventory
SSL=0
TAG=MonEntreprise

Installation de l'agent Linux

# Installer les dépendances
sudo apt install -y libmodule-install-perl dmidecode libxml-simple-perl \
  libcompress-zlib-perl libnet-ip-perl libwww-perl libdigest-md5-perl libdata-uuid-perl

# Télécharger et installer l'agent
cd /tmp
wget https://github.com/OCSInventory-NG/UnixAgent/releases/download/v2.10.0/Ocsinventory-Unix-Agent-2.10.0.tar.gz
tar -xvzf Ocsinventory-Unix-Agent-2.10.0.tar.gz
cd Ocsinventory-Unix-Agent-2.10.0
perl Makefile.PL && make && sudo make install

# Premier inventaire
sudo ocsinventory-agent --server http://IP_SERVEUR/ocsinventory --tag Linux-Servers

# Automatiser avec cron
echo "0 12 * * * root /usr/local/bin/ocsinventory-agent" | sudo tee /etc/cron.d/ocsinventory

Intégration GLPI ↔ OCS

Synchronisation automatique
  1. Dans GLPI : Configuration → Plugins
  2. Installer et activer le plugin OCS Inventory NG
  3. Configurer la connexion au serveur OCS
  4. Configurer les règles d'import
  5. Planifier la synchronisation automatique
