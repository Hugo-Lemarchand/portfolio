Migration Domaine vers Intune

Active Directory → Microsoft Entra ID + Microsoft Intune

Hugo Lemarchand · Ensitech · Session 2026

Présentation

Ce projet décrit la migration d'une infrastructure gérée par Active Directory on-premise vers une gestion cloud avec Microsoft Entra ID (ex-Azure AD) et Microsoft Intune (MDM).

Cette migration permet une gestion centralisée depuis le cloud, un accès aux applications depuis n'importe quel endroit, et une sécurité renforcée via l'accès conditionnel et le MFA.

Phase 4 – Migration des postes

10

Option A – Migration en place

# Vérifier l'état actuel
dsregcmd /status

# Quitter le domaine
Remove-Computer -UnjoinDomainCredential (Get-Credential) -PassThru -Verbose -Restart

# Après redémarrage : Paramètres → Comptes → Accès professionnel ou scolaire
# Choisir "Joindre cet appareil à Azure Active Directory"
11

Option B – Réinstallation avec Autopilot

  1. Enregistrer le poste dans Autopilot
  2. Réinitialiser Windows (Reset this PC)
  3. L'utilisateur se connecte avec ses identifiants M365
  4. Configuration automatique via Intune
12

Script de migration automatisé

$Credential = Get-Credential -Message "Compte admin domaine"

# Sauvegarder les données utilisateur
Copy-Item "$env:USERPROFILE\Desktop"   "D:\Backup\Desktop"   -Recurse
Copy-Item "$env:USERPROFILE\Documents" "D:\Backup\Documents" -Recurse

# Disjoindre du domaine
Remove-Computer -UnjoinDomainCredential $Credential -Force -Restart
⚠️ Attention Toujours tester sur un groupe pilote avant le déploiement massif !

Phase 5 – Configuration de la sécurité

13

Conditional Access

  • Exiger MFA pour les connexions à risque
  • Bloquer l'accès depuis des pays non autorisés
  • Exiger un appareil conforme pour accéder aux données
14

Compliance Policies

# Exemple de politique de conformité
- Version Windows minimum : 22H2
- BitLocker activé
- Antivirus actif
- Pare-feu activé
- Mot de passe requis

# Action si non conforme :
- Notification utilisateur
- Bloquer l'accès après X jours
15

Windows Hello for Business

Configurer l'authentification sans mot de passe :

  • Devices → Windows → Enrollment → Windows Hello for Business

Checklist post-migration

  • Les applications sont déployées
  • Les politiques de conformité sont actives
  • L'accès conditionnel fonctionne correctement

  • La documentation est à jour
  • L'ancien AD peut être décommissionné (si full cloud)
