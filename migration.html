☁️ Migration Domaine vers Intune

Active Directory → Microsoft Entra ID + Intune

📋 Vue d'ensemble du projet

Cette migration permet de passer d'une gestion traditionnelle Active Directory (on-premise) vers une gestion cloud avec Microsoft Entra ID (Azure AD) et Microsoft Intune pour le MDM.

Avantages de la migration

  • Gestion centralisée depuis le cloud
  • Accès aux applications de n'importe où
  • Sécurité renforcée (Conditional Access, MFA)
  • Déploiement et configuration automatisés
  • Réduction des coûts d'infrastructure
Phase 1

🔍 Audit et planification

1Inventaire de l'existant

  • Nombre de postes et utilisateurs
  • Applications déployées (GPO, scripts)
  • Imprimantes et partages réseau
  • Stratégies de groupe (GPO) en place
  • Certificats et configurations spécifiques

2Prérequis licences

Licence Fonctionnalités
Microsoft 365 Business Premium Intune, Entra ID P1, Defender
Microsoft 365 E3/E5 Intune, Entra ID P1/P2, fonctionnalités avancées
Intune Plan 1 (standalone) MDM uniquement

3Planifier la coexistence

Définir si vous optez pour :

  • Hybrid Join : postes joints à AD et Entra ID (transition progressive)
  • Cloud-only : migration complète vers Entra ID (recommandé pour nouveaux postes)
Phase 2

⚙️ Configuration Entra ID et Intune

4Configurer Entra ID Connect (si Hybrid)

# Télécharger Azure AD Connect depuis Microsoft
# Installer sur un serveur membre du domaine

# Configuration recommandée :
- Synchronisation de hash de mot de passe
- Filtrage par OU si nécessaire

5Configurer l'inscription automatique Intune

Dans le portail Entra ID :

  1. Mobility (MDM and MAM) → Microsoft Intune
  2. MDM user scope : Tous ou groupe spécifique
  3. MAM user scope : Aucun (ou selon besoin)

6Configurer Windows Autopilot

# Sur un poste de test, récupérer l'ID matériel
Install-Script -Name Get-WindowsAutoPilotInfo
Get-WindowsAutoPilotInfo -OutputFile C:\autopilot.csv

# Importer dans Intune :
# Devices → Windows → Enrollment → Devices (Autopilot)
Phase 3

📝 Migration des GPO vers Intune

7Analyser les GPO existantes

# Exporter les GPO pour analyse
Get-GPO -All | ForEach-Object {
    Get-GPOReport -Guid $_.Id -ReportType Html -Path "C:\GPO_Reports\$($_.DisplayName).html"
}

8Utiliser Group Policy Analytics

Dans Intune : Devices → Windows → Group Policy analytics

  • Importer les fichiers de sauvegarde GPO
  • Analyser la compatibilité avec Intune
  • Migrer les paramètres compatibles

9Créer les profils de configuration

Types de profils Intune :

  • Device restrictions : sécurité, fonctionnalités Windows
  • Configuration profiles : paramètres personnalisés
  • Endpoint security : antivirus, pare-feu, chiffrement
  • Compliance policies : conformité des appareils
Phase 4

💻 Migration des postes

Option A : Migration en place (Hybrid → Cloud)

# Disjoindre du domaine et joindre Entra ID
# En tant qu'administrateur local

# Vérifier l'état actuel
dsregcmd /status

# Quitter le domaine
Remove-Computer -UnjoinDomainCredential (Get-Credential) -PassThru -Verbose -Restart

# Après redémarrage, joindre Entra ID
# Paramètres → Comptes → Accès professionnel ou scolaire → Connecter
# Choisir "Joindre cet appareil à Azure Active Directory"

Option B : Réinstallation avec Autopilot

  1. Enregistrer le poste dans Autopilot
  2. Réinitialiser Windows (Reset this PC)
  3. L'utilisateur se connecte avec ses identifiants M365
  4. Configuration automatique via Intune

10Script de migration automatisé

# Script PowerShell de migration
$Credential = Get-Credential -Message "Compte admin domaine"

# Sauvegarder les données utilisateur si nécessaire
$UserProfile = $env:USERPROFILE
Copy-Item "$UserProfile\Desktop" "D:\Backup\Desktop" -Recurse
Copy-Item "$UserProfile\Documents" "D:\Backup\Documents" -Recurse

# Disjoindre du domaine
Remove-Computer -UnjoinDomainCredential $Credential -Force -Restart
⚠️ Attention : Toujours tester sur un groupe pilote avant le déploiement massif !
Phase 5

🔒 Configuration de la sécurité

11Conditional Access

Créer des stratégies d'accès conditionnel :

  • Exiger MFA pour les connexions à risque
  • Bloquer l'accès depuis des pays non autorisés
  • Exiger un appareil conforme pour accéder aux données

12Compliance Policies

# Exemple de politique de conformité
- Version Windows minimum : 22H2
- BitLocker activé
- Antivirus actif
- Pare-feu activé
- Mot de passe requis

# Action si non conforme :
- Notification utilisateur
- Bloquer l'accès après X jours

13Windows Hello for Business

Configurer l'authentification sans mot de passe :

  • Devices → Windows → Enrollment → Windows Hello for Business
  • Activer et configurer PIN, biométrie
Phase 6

📦 Déploiement des applications

14Apps Microsoft 365

# Dans Intune : Apps → Windows → Add
# Type : Microsoft 365 Apps

# Configuration recommandée :
- Update channel : Monthly Enterprise
- Apps : Word, Excel, PowerPoint, Outlook, Teams
- Architecture : 64-bit
- Version : Latest

15Applications Win32

# Préparer le package avec IntuneWinAppUtil
IntuneWinAppUtil.exe -c "C:\Source\App" -s "setup.exe" -o "C:\Output" -q

# Configurer dans Intune :
# Install command : setup.exe /silent
# Uninstall command : setup.exe /uninstall /silent
# Detection rules : fichier, registre ou script
