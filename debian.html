Installation Debian

Système d'exploitation GNU/Linux stable et fiable

Hugo Lemarchand · Ensitech · Session 2026

Présentation

Ce tutoriel détaille l'installation et la configuration initiale de Debian GNU/Linux, distribution reconnue pour sa stabilité et largement utilisée en environnement serveur. Debian constitue la base de nombreuses infrastructures SISR et est le système utilisé pour déployer la plupart des services de ce portfolio.

Prérequis matériels

ComposantMinimumRecommandé
RAM512 Mo2 Go+
Espace disque10 Go20 Go+
Processeur1 GHz2 GHz+
  • Clé USB de 4 Go minimum
  • Image ISO Debian (netinst recommandée) depuis debian.org

Étapes d'installation

1

Création de la clé USB bootable

# Windows : utiliser Rufus ou Balena Etcher
# Linux avec dd :
sudo dd if=debian-12.*-amd64-netinst.iso of=/dev/sdX bs=4M status=progress sync
2

Démarrage et sélection du mode

  • Booter sur la clé USB depuis le BIOS/UEFI
  • Choisir Graphical Install ou Install (mode texte)
3

Configuration de base

  • Langue : Français
  • Pays : France
  • Clavier : Français (azerty)
4

Configuration réseau

Nom de la machine : debian-server
Domaine : local (ou laisser vide)
5

Configuration des utilisateurs

  • Mot de passe root (laisser vide pour désactiver root et utiliser sudo)
  • Créer un utilisateur standard avec un mot de passe sécurisé
6

Partitionnement du disque

Pour les débutants : choisir Assisté – utiliser un disque entier → Tous les fichiers dans une seule partition.

💡 Configuration avancée recommandée 
/boot   500 Mo  (ext4)
/       20 Go   (ext4)
/home   Reste   (ext4)
swap    2× RAM  (max 8 Go)
7

Sélection des logiciels

  • Environnement de bureau (GNOME, KDE, XFCE…) — selon besoin
  • ✔ Serveur SSH
  • ✔ Utilitaires usuels du système
8

Installation de GRUB

  • Installer GRUB sur le disque principal
  • Sélectionner /dev/sda

Configuration post-installation

Mise à jour du système

sudo apt update
sudo apt upgrade -y
sudo apt full-upgrade -y

Installation des outils essentiels

sudo apt install -y vim curl wget git htop net-tools

Configuration de sudo

su -
usermod -aG sudo votre_utilisateur
# Déconnecter et reconnecter pour appliquer

Configuration SSH

sudo nano /etc/ssh/sshd_config

# Recommandations sécurité :
PermitRootLogin no
PasswordAuthentication yes
Port 22

sudo systemctl restart sshd

Configuration du pare-feu (UFW)

sudo apt install ufw
sudo ufw allow ssh
sudo ufw enable
sudo ufw status

Commandes essentielles

# Gestion des paquets
apt search <paquet>
apt install <paquet>
apt remove <paquet>
apt autoremove

# Informations système
uname -a
lsb_release -a
df -h
free -h

# Gestion des services
systemctl status <service>
systemctl start/stop/restart <service>
systemctl enable/disable <service>

Sécurisation de base

Bonnes pratiques
  1. Mettre à jour régulièrement le système
  2. Configurer les mises à jour automatiques de sécurité
  3. Désactiver les services inutiles
  4. Configurer fail2ban contre les attaques brute-force
sudo apt install fail2ban
sudo systemctl enable fail2ban
sudo systemctl start fail2ban
